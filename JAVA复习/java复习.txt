一.面向过程与面向对象的区别
	面向过程性能比面向对象高,由于类的调用实例化需要消耗较大的资源,所以当性能为开发首选时,比如单片机,linux,unix开发采用面向过程,但是面向过程没有面向对象那样的易维护,易复用,易扩展.
	面向对象性能比面向过程低,但是面向对象有分装,继承,多态等特性,可以开发出易维护,易复用,易扩展的低耦合程序,更加灵活.
二.java语言的特点
	1.面向对象,封装,继承,多态
	2.平台无关,(java虚拟机)
	3.可靠性,安全性
	4.支持多线程(c++是不在内部提供多线程机制的,所以需要调用操作系统的多线程功能来实现,java则提供了多线程)
	5.自动垃圾回收机制
	6.支持网络编程且比较方便,解释与编译并存
	
三.JVM JDK JRE
	JVM:java虚拟机,java运行字节码文件的虚拟机,JVM针对不同的系统有特定的实现(win,linux,macOS),目的是使用相同的字节码文件,都能够得出相同的结果
	字节码:JVM可以理解的代码称之为字节码文件(.class),他不面向任何指定的处理器,只面向虚拟机.java通过字节码的方式,一定程度上降低了解释型语言效率低的问题,同时有保留了解释性语言可移植的特点.		.java源文件(javac 编译) .class文件 (JVM) 机器可执行的二进制机器码
	.class-->机器码是JVM类加载器首先加载字节码文件,然后通过解释器逐行解释执行(这种方式的执行速度相对较慢),
